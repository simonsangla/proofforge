# Information Architecture

## Top-Level Pages
- `/` — Homepage
- `/lab` — Artifact index
- `/lab/[slug]` — Artifact detail page
- `/play/[slug]` — Public interactive demo
- `/replica/[slug]` — Limited replication page
- `/notes` — Best practices, code notes, prompt notes
- `/about` — Positioning, offers, contact

## Homepage Blocks
1. Hero: what ProofForge is
2. Current publishing velocity
3. Featured artifacts
4. Category navigator
5. Why it matters commercially
6. Contact / collaboration CTA

## Lab Page Responsibilities
- filter by category
- filter by status
- filter by protection level
- sort by newest, category, or signal
- surface featured items first

## Artifact Detail Page Responsibilities
- artifact promise
- business value
- visual proof
- stack
- status
- protection level
- live links
- explanation of what is public versus private

## Rules
- keep navigation shallow
- use one canonical slug per artifact
- avoid duplicate content across detail and replica pages
