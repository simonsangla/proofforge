# Weekly Publishing System

## Target
5 to 7 items per week.

## Recommended Mix
- 1 to 2 mini-apps
- 1 to 2 Prompt Dojo or prompt-related assets
- 1 code pattern or best-practice note
- 1 UI pattern or product experiment
- 1 optional wild-card item

## Cadence Logic
This is optimal because:
- mini-apps create depth but are expensive
- smaller artifacts preserve publishing velocity
- the mix compounds both execution proof and conceptual clarity

## Weekly Tracking Fields
- planned_items
- shipped_items
- carry_over
- strongest_signal
- reusable_assets_created

## Rule
Do not force 7 heavy items. Force 7 meaningful proofs.
